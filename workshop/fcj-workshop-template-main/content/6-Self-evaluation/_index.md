Learning step by step helped me understand better how an application is deployed to the AWS environment.

Besides the weekly labs, I also joined the team project **OCR-CapCut**. This project helped me understand how a real application needs to combine many parts such as backend, frontend, database, file storage, and video processing. While working on the project, I also practiced reading documentation, checking errors, discussing with teammates, and recording work progress more clearly.

### Achievements

At first, I was quite confused when working with IAM Policy, Security Group, and connecting services together. After practicing many times, I became more familiar with checking each step, reading errors, and reviewing the configuration.

* Understood more clearly how IAM is used to manage access and protect an AWS account.
* Learned how to deploy a basic web app on EC2 and combine it with Auto Scaling and Application Load Balancer.
* Used Route 53, S3, and CloudFront to manage domains, store static files, and deliver content over HTTPS.
* Gained more experience with RDS MySQL, Docker, ECR, and ECS during application deployment.
* Learned how to monitor costs with AWS Budgets and Cost Explorer, and control usage based on the $200 AWS Free Tier credits.
* Completed the worklog, report, and assigned tasks in the OCR-CapCut team project.

### Areas for improvement

* Practice drawing the architecture before deployment to avoid missing steps or making configuration mistakes.
* When errors occur between ECS, RDS, S3, or IAM, I sometimes still spend a lot of time identifying whether the error is in access permissions, network, or service configuration.
* Present technical problems more briefly when discussing with the team.
* Review networking, security groups, DNS, and certificates more carefully.

### Direction after the internship

After the internship, I want to continue practicing deploying a complete application on AWS, from backend, database, storage to containers. I will also spend more time learning about networking, security groups, IAM policies, and CI/CD because these are the areas I still need to improve.
