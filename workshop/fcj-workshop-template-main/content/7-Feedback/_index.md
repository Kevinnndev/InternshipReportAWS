After joining FCAJ, I found the program quite suitable for students who are new to cloud. The content did not only focus on theory but also included many labs for practicing directly on AWS Console.

### Overall evaluation of the program

**1. Training content**

The learning path was divided by week, so it was easy to follow. In the first weeks, the program focused on basic knowledge such as Cloud Computing, IAM, and cost management. The later weeks moved to application deployment with EC2, Auto Scaling, Application Load Balancer, Route 53, S3, CloudFront, RDS, Docker, ECR, and ECS. Learning step by step like this helped me connect the topics more easily.

**2. Practice part**

The practice part was the most useful part for me. When I created IAM Users, configured EC2, created S3 buckets, configured CloudFront, or ran containers on ECS by myself, I understood more clearly why each service is used in a system. Checking and deleting resources after the labs also made me pay more attention to AWS costs.

**3. Support from mentors and organizers**

During the program, the mentors and organizers supported us well. When I had errors, the mentor usually suggested what to check so I could find the cause by myself instead of giving the answer immediately. For example, when I had errors with S3 access permissions or security groups, the mentor guided me to check the policy, inbound rules, and logs instead of fixing it directly. This helped me understand the errors better and feel more confident when I saw similar issues again.

**4. Relevance to my major**

For Software Engineering students, this program was quite suitable because the topics were related to application deployment. I did not only learn about cloud infrastructure, but also understood more about how backend, frontend, database, storage, and security are connected in a project.

**5. OCR-CapCut team project**

The OCR-CapCut team project helped me apply the knowledge from the program to a specific problem. Because the project had many related parts, the team needed to agree on API, database schema, file upload flow, and how to store the processing results. This part helped me see the importance of discussing before coding. During the work, I learned more about task division, progress updates, and preparing documents for the demo.

### What I was most satisfied with

The thing I was most satisfied with was that after the program, I had a clearer view of how to deploy an application on AWS. Before that, I only knew some cloud concepts at a basic level. After studying and doing labs, I understood more about how services work together, from access control, running servers, storing data, and delivering content to deploying containers.

Besides technical knowledge, the program also helped me practice recording progress, asking clearer questions, and discussing with the team when facing problems. These are skills I find important when working in a real environment.

### Suggestions for improvement

* A short checklist after each lab could be added so learners can review created resources and avoid forgetting to delete resources.
* More time could be spent on common errors, especially IAM permission, Security Group, ECS task log, RDS connection, and CloudFront cache issues.
* It would be more helpful if there was an architecture review session before the team project so teams could agree on the scope, AWS services to use, and task division more clearly.
* More final report criteria could be provided so students can prepare more focused content.

### Expectations after the program

After the program, I want to join more advanced AWS workshops, especially about containers, security, CI/CD, and system architecture. These are the parts I think I need to learn more to work better in real projects.

### Conclusion

Overall, First Cloud AI Journey is a useful internship program for students who are new to cloud. The program helped me build basic AWS knowledge, practice self-learning and teamwork, and understand more clearly the direction of application deployment on cloud.
